# Гайд по публикации

## Рекомендуемая схема

1. Основная площадка: GitHub (репозиторий + Releases)
2. Зеркало: GitLab или Codeberg
3. Посты в сообществах: Reddit/Telegram/форумы только со ссылкой на репозиторий

## Почему так

- Люди видят исходники и историю коммитов.
- Удобно проверять хэши релизов.
- Проще поддержка через Issues.

## Что класть в релиз

На каждый релиз:

- zip-архив
- `SHA256SUMS`
- `SIGNATURES`
- changelog

## Процесс релиза

1. Обновите версию.
2. Соберите:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\maintainer\Build-Release.ps1 -Version v1.0.0
```

3. Загрузите артефакты из `dist` в GitHub Release.
4. Добавьте раздел с командами проверки хэшей.

## Текст доверия (можно копировать)

> Открытый исходный код.  
> Проверяйте SHA256 перед запуском.  
> Скрипт не запрашивает Apple ID/пароль.  
> Логи не отправляются на сторонние сервисы.  
> Сеть используется только для официальных URL Sideloadly/Apple.

## Чего избегать

- публикации только `.exe` без исходников
- сторонних зеркал без проверяемых хэшей
- вшитых DLL без объяснения источника
